The questions every owner asks

The honest answers — because trust is the whole point.

Where is my data stored?

On your own device, in the browser — never on our servers. If you run several registers you can sync them through a small server that you own. We never see your customers or your sales.

Is it backed up? What if my computer dies?

Your data is plain JSON you can export any time. Run the optional hub and you get rolling snapshots and an off-site copy. You're never one hard drive away from losing everything.

Does it work if the internet goes down?

Yes. It runs entirely from local data and re-syncs when you're back online. An outage never stops you ringing up a sale — a real advantage over cloud-only systems.

Am I locked in? Can I get my data out?

No lock-in. It's one HTML file and plain-JSON data, both yours. Export whenever you like. There's nothing proprietary to trap you.

What does it cost to run?

The software is $0 forever. A sync server, if you want one, is a few dollars a month. Texting customers costs only if you use it. Card processing is a fee you already pay — and the only place we earn.

How long until I'm up and running?

Download the file and open it in Chrome — it works immediately. Configuring your prices and importing data are guided steps, not a multi-week consultant project.

Who helps me when I'm stuck or want changes?

Every download ships with a CLAUDE.md guide. Open your file in Claude Code and change prices, receipts, or whole workflows in plain English — on your own machine.

What if customPOS disappears?

You own the code and it runs locally, so it keeps working with zero dependency on us. That's the strongest promise we can make — and the reason it's built this way.
